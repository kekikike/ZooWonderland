# ZooWonderland - Sistema de Gestión de Zoologico

# Objetivo del Proyecto
El objetivo del sistema se basa en mejorar los procesos relacionados con las ventas
de entradas y la gestion de recorridos guiados.

Justificando que actualmente el Zoologico realiza la parte de administrador usando
metodos en papel y limita el acceso de los usuarios

# Metodologia de trabajo
Se usa la metodologia agil Extreme Programming (XP)
aplicando:
- Desarrollo iterativo en ciclos cortos
- POO
- Retroalimentacion constante
- Pruebas continuas
- Refactorizacion del codigo

# Backlog
Se tienen las siguientes funcionalidades:
- Gestión de animales
- Gestión de empleados
- Control veterinario
- Control de alimentación
- Venta de entradas
- Reportes administrativo

# Plan de Iteraciones

# Iteracion I - Modulo base
Duracion: 5 Dias
Objetivo: Desarrollar el sistema con el modulo del cliente funcional
incluyendo:
- Login y Registro
- Index
- Comprar entradas
- Reservar tours
- ver historial de compras

# Iteracion II - Modulo Guia
Duración: 4 dias
Objetivo: Implementar la gestión de recorridos guiados.

Incluye:
- Registro de guías
- Asignación de tours
- Visualización de horarios
- Reporte de recorridos realizados

# Iteracion III - Modulo Adminsitrador
Duración: -- dias
Objetivo: Implementar la gestión administrativa del zoológico.

Incluye:
- Gestión de empleados
- Gestión de animales
- Gestión de reportes
- Visualización de estadísticas
- Administración de usuarios

# Criterios de aceptacion
El desarrollo de la primera iteracion es exitosa cuando se logran realizar todas las operaciones
en Programacion Orientada a objetos sin ningun error critico.

# Riesgos identificados
Se detecto la falta de una base de datos SQL para el control eficiente y eficaz del registro y 
manipulacion de datos
